title: Setting Envelope Item
description: Step-by-step guide to configure physical envelope items
---

This guide walks you through setting up a custom item for physical lucky money envelopes. Physical envelopes allow players to create tradeable items that hold money.

---

## Prerequisites

Pre-Made Configs:

- [ItemsAdder-Config](https://drive.google.com/file/d/1yicc8VdExr9TNkldN_4xDoTesc4zVloe/view?usp=sharing)
- [Nexo-Config](https://drive.google.com/file/d/1XFpDTnJ_wRuE8WyF81pJywfKhJ2zvfM3/view?usp=sharing)

Before you can use physical envelopes, you need:

1. **A custom item plugin** - One of:
   - [ItemsAdder](https://www.spigotmc.org/resources/73355/)
   - [Oraxen](https://www.spigotmc.org/resources/72448/)
   - [Nexo](https://www.spigotmc.org/resources/116878/)

2. **A custom item created** - Design or download a red envelope item in your item plugin

3. **Admin permission** - `lixi.admin` permission

---

## Step 1: Create a Custom Item

First, create a custom item in your item plugin. This will be the physical envelope that players receive.

### ItemsAdder Example

Edit your ItemsAdder config (e.g., `items/lixi.yml`):

```yaml
info:
  namespace: lixi

items:
  red_envelope:
    display_name: "<gradient:#ff0000:#ffaa00>Red Envelope</gradient>"
    permission: "lixi.use"
    resource:
      material: PAPER
      generate: true
      model_path: item/red_envelope
    specific_properties:
      note_block:
        custom_model_data: 12001
```

Then run:
```bash
/iazip
/iareload
```

### Oraxen Example

Edit your Oraxen config (e.g., `items.yml`):

```yaml
red_envelope:
  displayname: "<gradient:#ff0000:#ffaa00>Red Envelope</gradient>"
  material: PAPER
  Pack:
    generate_model: true
    model: red_envelope
    custom_model_data: 12001
```

Then run:
```bash
/oraxen reload all
```

### Nexo Example

Edit your Nexo config (e.g., `items/envelopes.yml`):

```yaml
red_envelope:
  type: ITEM
  material: PAPER
  display_name: "<gradient:#ff0000:#ffaa00>Red Envelope</gradient>"
  model: 12001
```

Then run:
```bash
/nexo reload
```

---

## Step 2: Get the Item

Use your item plugin's give command to obtain the custom item:

```bash
# ItemsAdder
/ia get red_envelope

# Oraxen
/o give red_envelope

# Nexo
/nexo give red_envelope
```

You should now have the custom item in your inventory.

---

## Step 3: Run the SetItem Command

1. **Hold the custom item** in your main hand (not offhand)
2. **Run the command:**

```bash
/lixiadmin setitem
```

3. **Check the output:**

If successful, you'll see:
```
Detected: itemsadder:red_envelope
Set ItemsAdder item: red_envelope
[Lì Xì] » Đã đặt vật phẩm lì xì thành công!
```

The plugin automatically:
- Detects which plugin the item is from
- Extracts the item ID
- Saves it to `main-config.yml`
- Clears any other plugin IDs to prevent conflicts

---

## Step 4: Verify Configuration

Check your `main-config.yml` - it should now show:

```yaml
envelope:
  itemsadder:
    item-id: "red_envelope"  # ✓ Your item ID
  oraxen:
    item-id: ""              # ✗ Cleared
  nexo:
    item-id: ""              # ✗ Cleared
  display-name: "<color:#e80c00>Lì Xì</color>"
  lore:
    - "<gray>Chứa <gold>%amount%</gold></gray>"
    - "<gray>Click chuột phải để nhận tiền!</gray>"
```

:::tip[Display Name Override]
The `display-name` and `lore` settings in the config will **override** the item's default name and lore when envelopes are created. You can customize these to match your server's language or theme.
:::

---

## Step 5: Test It

Create a test envelope to verify everything works:

```bash
/lixi phongbao 1000
```

You should receive a custom envelope item with:
- Your custom model/texture
- The configured display name
- Lore showing the amount (1,000)

Right-click the envelope to claim the money!

## Troubleshooting

### "Bạn phải cầm một vật phẩm trong tay!"

**Problem:** You're not holding anything, or the item is in your offhand.

**Solution:** Make sure the custom item is in your **main hand** (the slot you're actively holding).

---

### "Could not detect item type!"

**Problem:** The item is vanilla (not from a custom item plugin) or is unsupported.

**Solution:**
- Verify you're holding a custom item from ItemsAdder/Oraxen/Nexo
- Check that the custom item plugin is installed and loaded
- Try `/ia get <item>` (or equivalent) to get the item fresh

---

### "Unsupported plugin type: XYZ"

**Problem:** The item is from a plugin that LiXiPlugin doesn't support.

**Solution:**
- Only **ItemsAdder**, **Oraxen**, and **Nexo** are supported
- Request support for other plugins on [GitHub Issues](https://github.com/SimpMC-Studio/LiXiPlugin/issues)

---

### Envelope Creates But Shows Wrong Item

**Problem:** The setitem command worked, but `/lixi phongbao` gives a different item.

**Solution:**
1. Check which custom item plugin is actually installed
2. Verify the item ID is correct in `main-config.yml`
3. Make sure only ONE plugin's `item-id` is set (others should be `""`)
4. Run `/lixiadmin reload` after manual config edits

---

## Changing the Envelope Item

To switch to a different envelope item:

1. Get the new custom item
2. Hold it in your main hand
3. Run `/lixiadmin setitem` again

The plugin will automatically clear the old item ID and save the new one.

---

## Custom Display Names & Lore

You can customize how envelopes appear in `main-config.yml`:

```yaml
envelope:
  display-name: "<gradient:#ff0000:#ffaa00>Lucky Red Envelope</gradient>"
  lore:
    - "<gray>Contains <gold>%amount%</gold></gray>"
    - "<yellow>Right-click to claim!</yellow>"
    - ""
    - "<dark_gray>From YourServerName</dark_gray>"
```

**Available Placeholders:**
- `%amount%` - The money amount in the envelope

All text supports [MiniMessage](https://docs.advntr.dev/minimessage/format.html) formatting.

After editing, reload:
```bash
/lixiadmin reload
```
